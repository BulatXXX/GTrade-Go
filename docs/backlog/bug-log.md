# Bug Log

Журнал ошибок, дефектов и аномалий в системе GTrade.

Этот файл используется только для багов.

Для задач улучшения и следующих шагов используется:

- `docs/backlog/roadmap.md`

## Как писать записи

Для каждой ошибки фиксировать:

- `ID`
- `Дата`
- `Компонент`
- `Серьезность`
- `Статус`
- `Как воспроизвести`
- `Ожидаемое поведение`
- `Фактическое поведение`
- `Примечание`

Рекомендуемые статусы:

- `new`
- `confirmed`
- `in_progress`
- `fixed`
- `wont_fix`

Рекомендуемые severity:

- `critical`
- `high`
- `medium`
- `low`
