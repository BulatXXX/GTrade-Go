# Техническое задание
на разработку системы для управления данными внутриигровых торговых площадок для обеспечения игрового прогресса пользователя

---

## 1 Введение

### 1.1 Наименование программы
Веб-ресурс для управления данными внутриигровых торговых площадок для обеспечения игрового прогресса пользователя

### 1.2 Область применения

Разрабатываемая система применяется в области обработки и анализа данных внутриигровых торговых площадок различных онлайн-игр.

Система используется в рамках веб-приложений и предназначена для работы с данными о предметах, их характеристиках и стоимости, получаемыми из внешних источников.

### 1.3 Объект применения

Объектом применения является процесс анализа данных внутриигровых торговых площадок пользователями с целью повышения эффективности игрового прогресса.

Система используется конечными пользователями через веб-интерфейс, а также может использоваться как программный интерфейс для интеграции с другими программными компонентами.
## 2 Основания для разработки

Разработка выполняется в рамках учебного проекта по дисциплине «Технологии создания программного обеспечения».

Необходимость разработки обусловлена сложностью анализа данных внутриигровых торговых площадок и отсутствием удобного инструмента для их обработки в едином интерфейсе.

---

## 3 Назначение разработки

### 3.1 Функциональное назначение
Система должна обеспечивать:
- получение и обработку данных о предметах из внутриигровых торговых площадок
- хранение локального каталога предметов
- предоставление информации о предметах и их стоимости
- поддержку пользовательских сценариев для анализа и отслеживания предметов

### 3.2 Эксплуатационное назначение
Система предназначена для использования:
- конечными пользователями через веб-интерфейс
- разработчиками через API

---

## 4 Требования к программе

### 4.1 Требования к функциональным характеристикам

Система должна реализовывать следующие функции:

#### 4.1.1 Работа с пользователями
- регистрация и авторизация
- управление профилем
- восстановление доступа

#### 4.1.2 Работа с каталогом
- хранение предметов
- поиск предметов
- получение карточки предмета

#### 4.1.3 Работа с данными торговых площадок
- получение актуальных данных о ценах
- нормализация данных
- обновление информации

#### 4.1.4 Пользовательские данные
- список отслеживаемых предметов
- история просмотров
- пользовательские настройки

#### 4.1.5 Уведомления
- отправка email
- уведомления об изменении параметров предметов

---

### 4.2 Требования к надежности

Система должна обеспечивать:
- устойчивость к отказам внешних источников данных
- корректную обработку ошибок
- ведение логов

---

### 4.3 Требования к условиям эксплуатации

Система должна функционировать:
- в контейнерной среде Docker
- на сервере с операционной системой Linux

---

### 4.4 Требования к составу и параметрам технических средств

Минимальные требования:
- CPU: 2 ядра
- RAM: 4 ГБ
- Диск: 10 ГБ

---

### 4.5 Требования к информационной и программной совместимости

Используемые технологии:

Frontend:
- Angular
- Angular Material

Backend:
- Go
- Gin

База данных:
- PostgreSQL

Инфраструктура:
- Docker
- GitHub

- User Asset Service

- Notification Service

Система использует внешние API внутриигровых торговых площадок.

- получение информации о предмете

- получение информации о предметах
- получение цен
- использование статического каталога данных

Escape from Tarkov:
- получение информации о предметах
- получение цен
- поиск предметов
